# Product Documentation

## Purpose
Define SNTO's product identity, users, workflows, commercialization path, and feature strategy.

## Background
The CPO and protected-area director reviews evaluated SNTO as a product rather than as science or code.

## Current State
SNTO is best positioned as protected-area decision intelligence, not as a generic GIS, BI dashboard, or academic observatory.
