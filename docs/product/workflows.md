# Product Workflows

## Purpose
Define the core workflows SNTO should support.

## Background
The reviews repeatedly found that SNTO must move from dashboard to accountable workflow.

## Current State
Core workflows:
1. Weekly risk review.
2. Field verification.
3. Budget planning.
4. Visitor management.
5. Evidence review.
6. Public and political reporting.

## Evidence
The protected-area director review said the home page should answer what needs attention, why, confidence, recommended action, cost, owner, and deadline.

## Recommendations
Make managed asset state the product core: detected -> verified -> assigned -> funded -> resolved -> monitored.

## Next Steps
Turn workflows into roadmap requirements for v1.2 and v2.0.
