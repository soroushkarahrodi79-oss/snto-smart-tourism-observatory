# SNTO v2.0.0 — Release notes

**Tag:** `v2.0.0` · **Commit:** `aad520b` · **Fecha:** 2026-07-20 ·
**Tests:** 821 recolectados (820 passing + 1 skipped) en el tag.
Pista de publicación: **A — artefacto de software** (ver
[`../PUBLICATION_STRATEGY.md`](../PUBLICATION_STRATEGY.md)).

> Esta es una release de **arquitectura, backend y UI** — la metodología
> científica es idéntica a la baseline de rigor estadístico v1.3.0. No introduce
> ni afirma validación de campo (campaña #26 pendiente).

## Titular

De un prototipo académico de un solo panel a una **capa de decisión modular,
con backend persistente y una interfaz reorganizada por roles** para gestores de
espacios naturales protegidos. Consuma la visión "v2.0" de la hoja de ruta
(arquitectura modular + fundamentos de backend + evolución de UI por roles).

## Qué trae v2.0.0 (sobre la base ya publicada en v1.5.0)

- **Evolución de UI por roles (Fase 6):** la información se reorganiza en
  **cuatro capas de decisión** — Decidir · Diagnosticar · Evidenciar ·
  Gobernar — con *home* por audiencia (Técnica / Gestor / Auditoría), **el
  activo como página**, triaje de alertas, y módulos de simulación de
  presupuesto, presión/capacidad, confianza, proveniencia, informes/exportaciones
  y configuración territorial.

## Qué ya estaba en v1.5.0 y v2.0.0 consolida

- **Arquitectura modular (Fase 4, #27):** `app.py` de ~3.170 → ~285 líneas
  (solo composición); UI extraída a `src/ui/`.
- **Vistas por audiencia (#28):** Técnica / Gestor / Auditoría con divulgación
  por capas y cifras financieras invariantes entre vistas.
- **Backend persistente (Fase 5, ADR-011):** SQLAlchemy 2.0 + Alembic,
  repositorios tipados, `/api/v2` de lectura+escritura, máquinas de estado de
  ciclo de vida, rastro de auditoría y auth mínima por API-key. **Producción
  sobre Azure PostgreSQL** desde el cutover 2026-07-18. (El `/api/v2` existe
  como código + tests; no desplegado como servicio, ADR-012.)

## Camino recorrido v1.0.0 → v2.0.0 (para contexto)

| Release | Aporte |
|---|---|
| v1.1.x | Serie temporal Sentinel-2 real 2021–2026 (GEE); Mann-Kendall desestacionalizado + Yue-Pilon |
| v1.2.0 | Piloto de replicabilidad Red OAPN (Tablas de Daimiel, Monfragüe) |
| v1.3.0 | Rigor estadístico (Pettitt, bootstrap IC, Morris, cross-sensor MODIS) |
| v1.4.0 | Integración para decisión (risk brief, export GIS, gating de evidencia, herramientas de validación) |
| v1.5.0 | Modularización (#27) + vistas por audiencia (#28) + backend persistente (Fase 5) |
| **v2.0.0** | **Evolución de UI por roles (Fase 6)** — 4 capas de decisión |

## No negociables (siguen intactos)

- Separación de evidencia real / calibrado / sintético / simulado (ADR-004).
- Sin sobre-afirmar validez científica antes de la validación de campo (ADR-003).
- La campaña de campo del PNSG (#26) **sigue pendiente**: "tendencia satelital
  real" ≠ "validado en campo".

## Ya en desarrollo tras v2.0.0 (aún sin nueva release estable)

`main` (`2.1.0.dev0`) ya incorpora: **v2.1** (activación/gobernanza — deploy
gateado por CI, CI endurecido), **v2.2** (forecasting `SIMULATED`, movilidad
real MITMA, capacidad LAC/ROS, zonas SCM reales), el **runner de la puerta de
