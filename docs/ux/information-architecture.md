# Information Architecture

## Purpose
Define the future information architecture for SNTO.

## Background
The reviews found that SNTO's current information hierarchy is too broad and explanatory.

## Current State
Operational users need separated experiences:
- director overview;
- operations/action console;
- analyst evidence view;
- field verification workflow;
- reporting and audit center.

## Evidence
The CPO review recommended making "managed asset with evidence and action state" the core object, not dashboard tabs.

## Recommendations
Authoritative hierarchy:
1. Asset.
2. Risk.
3. Confidence.
4. Cause hypothesis.
5. Recommended action.
6. Cost.
7. Owner/status.
8. Evidence.
9. Report.

## Next Steps
Use this hierarchy for v2.0 IA planning.
