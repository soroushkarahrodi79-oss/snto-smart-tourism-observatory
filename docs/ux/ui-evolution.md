# UI Evolution

## Purpose
Define the long-term UI evolution direction.

## Background
The consensus review concluded that SNTO must evolve from observatory interface to accountable decision system.

## Current State
The current UI is a strong demonstration surface, not yet an operational institutional console.

## Evidence
Multiple reviews stated that SNTO should show less machinery and more decision workflow.

## Recommendations
Evolution path:
1. Decision-first home page.
2. Role-specific views.
3. Evidence-on-demand.
4. Field verification workflow.
5. Report generation.
6. Audit center.

## Next Steps
Tie UI evolution to v1.2 and v2.0 product roadmap.
