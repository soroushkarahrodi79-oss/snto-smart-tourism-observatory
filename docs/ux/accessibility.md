# Accessibility

## Purpose
Record accessibility concerns and requirements.

## Background
The repository audit identified risks around color dependence, dense custom HTML, and chart/map accessibility.

## Current State
Accessibility is not proven. No formal accessibility QA process was identified in the reviews.

## Evidence
The audit recommended non-color labels, contrast checks, keyboard QA, alt equivalents for maps/charts, and reduced-density mode.

## Recommendations
Treat accessibility as a public-sector adoption requirement.

## Next Steps
Add accessibility acceptance criteria to v2.0 product work.
