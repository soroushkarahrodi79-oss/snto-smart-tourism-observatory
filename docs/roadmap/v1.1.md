# v1.1 Roadmap

## Purpose
Define v1.1 as the integration and stabilization release after the strategic review.

## Background
v1.1 should strengthen evidence clarity and product focus without risking v1.0 stability.

## Current State
Current SNTO is broad and demonstration-heavy. v1.0 should be treated as already deployed; v1.1 should not become a redesign branch.

## Evidence
Reviews called for product narrowing, real/calibrated/synthetic separation, director-grade risk brief, and validation planning.

## Recommendations
v1.1 priorities:
- clarify positioning around protected-area decision intelligence;
- improve separation of real, calibrated, synthetic, and simulated evidence;
- define the validation protocol and minimum evidence gates;
- create a lightweight director-grade risk brief prototype;
- stabilize branch integration and documentation alignment;
- avoid broad design exploration, enterprise backend work, or multi-territory expansion.

## Next Steps
Use v1.1 as the transition from observatory framing to decision-intelligence framing while preserving v1.0 behavior.
