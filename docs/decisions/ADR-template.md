# ADR Template

## Purpose
Provide the required format for future architecture decision records.

## Background
ADRs preserve why decisions were made.

## Current State
Use this template for new ADRs.

## Evidence
Required fields come from the documentation architecture brief.

## Problem
Describe the decision problem.

## Context
Summarize relevant background.

## Options Considered
List alternatives.

## Decision
State the chosen option.

## Consequences
Describe positive and negative consequences.

## Status
Proposed, accepted, superseded, or rejected.

## Related Documents
Link relevant docs.

## Future Review Date
Set a review date.

## Recommendations
Keep ADRs concise and decision-focused.

## Next Steps
Copy this file for new ADRs.
