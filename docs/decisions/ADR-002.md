# ADR-002: Build Toward Enterprise Platform Architecture

## Purpose
Record the enterprise architecture direction.

## Background
The CTO review found the current repository unsuitable for enterprise customers.

## Current State
SNTO is a pilot-grade analytical application, not an enterprise SaaS platform.

## Evidence
Security scored 25/100 and enterprise readiness 28/100 in the CTO review.

## Problem
Institutional adoption requires auth, authorization, audit, tenancy, persistence, observability, and release governance.

## Context
Enterprise protected-area customers need accountability and operational reliability.

## Options Considered
- Continue with Streamlit-first deployment.
- Add incremental patches to the current shell.
- Build a platform architecture around assets, users, territories, recommendations, interventions, and audit events.

## Decision
Future enterprise work should move toward platform architecture, with Streamlit treated as pilot/demo surface until replaced or isolated.

## Consequences
Positive: institutional readiness and scalable workflows. Negative: significant engineering effort before enterprise revenue.

## Status
Accepted from 2026 CTO and consensus reviews.
