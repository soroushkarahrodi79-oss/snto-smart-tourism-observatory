# ADR-010: Use a Hybrid Documentation Repository Strategy

## Purpose
Record the documentation repository separation decision.

## Background
The strategic knowledge base now includes product, scientific, commercial, institutional, and enterprise-readiness material that may grow beyond code documentation.

## Current State
The knowledge base currently lives inside the main repository.

## Evidence
The QA review identifies tradeoffs between repository cleanliness, confidentiality, collaboration, and relation to source code.

## Problem
Pure in-repo documentation keeps code and strategy close, but may expose sensitive strategic material and clutter the engineering repository as the knowledge base grows.

## Context
SNTO needs both technical docs near the code and strategic docs that may be shared with supervisors, institutions, advisors, or partners.

## Options Considered
- Keep all documentation in the main repository.
- Move all documentation to a separate knowledge-base repository.
- Use a hybrid model.

## Decision
Use a hybrid model unless a human stakeholder decides otherwise: keep code-coupled docs in the main repo and move strategic, commercial, review, and institutional knowledge to a separate `snto-knowledge-base` repository when collaboration/confidentiality needs grow.

## Consequences
Positive: preserves engineering clarity while enabling strategic documentation growth. Negative: requires cross-repo linking and documentation governance.
