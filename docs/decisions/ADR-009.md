# ADR-009: Formalize Deployment Governance Before Institutional Rollout

## Purpose
Record the deployment strategy decision.

## Background
The repository audit and CTO review identified deployment governance risk.

## Current State
SNTO has deployment mechanics but not enterprise-grade release governance.

## Evidence
Reviews identified inconsistency between CI-gated deployment claims and deploy behavior, plus missing staged promotion, rollback, observability, and release approvals.

## Problem
Institutional customers require predictable, auditable, reversible releases.

## Context
Protected-area agencies may rely on outputs for public reporting, budgets, and management decisions.

## Options Considered
- Continue direct deploy from main.
- Use manual deployment only.
- Establish staged environments, CI gates, smoke tests, release promotion, and rollback policy.

## Decision
Institutional rollout requires formal deployment governance with dev/staging/production separation and CI-gated release promotion.

## Consequences
Positive: reduces operational and reputational risk. Negative: increases DevOps maturity requirements.

## Status
Accepted from 2026 CTO and repository audit findings.
